# component - UI组件与视觉增强

本目录包含负责Neovim用户界面的组件和插件配置。这些插件专注于提升用户交互体验，提供直观的操作方式和美观的视觉界面。

## 插件说明

### oil.lua - 文件浏览器
Oil是一个高效的文件浏览器，提供了vim风格的文件操作体验。与传统的树形文件浏览器不同，Oil以编辑缓冲区的方式显示文件列表，用户可以像编辑文本一样操作文件：
- 直接在列表中选中、删除、重命名、移动文件
- 支持撤销和重做操作
- 可以编辑路径来快速导航目录
- 高效的快捷键操作，无需鼠标

Oil的这种设计使得文件操作更加高效，特别适合Vim用户的工作习惯。

### picker.lua - 选择器配置
配置Telescope选择器插件，提供了交互式的列表搜索界面：
- 快速搜索和过滤文件、缓冲区、符号等
- 实时预览选中项的内容
- 完全键盘驱动的操作方式
- 高度可自定义的外观和行为

该组件是项目查找和导航的核心，提高了信息搜索的效率。

### terminal.lua - 内置终端
集成内置的终端窗口，提供了：
- 在编辑器内运行shell命令
- 代码执行和调试输出
- 与Neovim的紧密集成

终端可以在新窗口或浮动窗口中打开，支持多个终端窗口并行运行。

## 组件特点

这三个组件的共同特点：
- **vim风格** - 所有操作遵循vim的按键习惯和操作逻辑
- **键盘驱动** - 完全支持键盘操作，无需鼠标
- **轻量级** - 不引入过度的依赖和功能膨胀
- **高效能** - 优化了性能和响应速度

## 与其他组件的协作

这些UI组件与其他部分的关系：
- 与config/keymaps配合 - 为选择器和文件浏览器提供快捷键
- 与plugins/util配合 - 使用工具函数进行配置
- 与utils配合 - 使用高亮和其他辅助工具

## 设计理念

项目不采用传统IDE风格的复杂UI布局，而是遵循以下原则：
- **极简主义** - 只提供必要的UI元素
- **高效性** - 优先考虑操作效率而非视觉华丽
- **vim一致性** - 保持与vim生态的兼容性和习惯

这种设计使得Neovim成为真正的编辑器而不是试图模仿IDE。
