# component — UI 交互组件插件

提供与日常编辑流程直接交互的核心 UI 组件，设计上遵循 vim 键盘驱动原则，避免引入鼠标依赖或 IDE 式的重型界面。

## 文件

- `oil.lua` — 以可编辑缓冲区形式呈现文件系统，像编辑文本一样完成文件的重命名、移动、删除、创建。支持撤销，与 yazi 等外部工具互补，日常文件操作优先使用 oil。
- `picker.lua` — Telescope 及 Snacks picker 的统一配置，包括外观、布局、预览行为和自定义 action。项目中所有模糊搜索入口（文件、符号、缓冲区、命令等）均经由此配置。
- `terminal.lua` — 内置终端窗口管理，支持浮动和分割两种布局，提供快速打开/关闭和多终端切换的快捷键，用于在不离开编辑器的情况下执行 shell 命令。

## 设计取向

本配置不使用 bufferline（用标签页代替）、不将文件树作为主要导航入口（用 picker 和 harpoon/grapple 代替），oil 和 picker 是文件操作的两个主要途径。
