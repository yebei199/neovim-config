# langs - 语言特定配置

本目录包含各种编程语言的特定配置。这里的配置与根目录`/langs`目录相互配合，但这里专注于在配置加载阶段需要的语言相关设置。

## 与根目录/langs的关系

项目的语言配置分为两层：
- **根目录/langs/** - 定义每种语言的LSP、格式化、Treesitter等完整工具链配置
- **lua/config/langs/** - 在Neovim配置加载时应用的额外语言相关设置

两个目录协同工作，提供了完整的语言支持系统。

## 文件说明

本目录包含以下语言的配置文件：
- **cpp.lua** - C++开发配置
- **python.lua** - Python开发配置
- **rust.lua** - Rust开发配置
- **lua.lua** - Lua开发配置
- **javascript.lua** - JavaScript/TypeScript开发配置
- **nix.lua** - Nix表达式语言配置
- **markdown.lua** - Markdown编写配置
- **html.lua** - HTML/Web开发配置
- **fish.lua** - Fish shell脚本配置
- **haskell.lua** - Haskell函数式编程配置
- **qml.lua** - Qt/QML配置

## 配置内容

每个语言配置文件通常包括：

### LSP配置
- 指定使用的语言服务器
- LSP服务器的特定选项和参数
- 代码补全、转到定义等功能的定制

### 格式化设置
- 代码格式化工具的选择和配置
- 缩进、行长等格式化选项
- 保存时自动格式化的规则

### Treesitter配置
- 指定Treesitter解析器
- 高亮和代码折叠的定制
- 文本对象的自定义

### 调试支持
- 调试器的配置（如dap配置）
- 断点、表达式求值等调试功能

### 插件集成
- 语言特定的Neovim插件
- 集成开发工具（如包管理器、构建系统等）
- 文档查看和帮助系统

## 配置模式

项目采用统一的配置模式，使得：
- 添加新语言支持时只需创建新的配置文件
- 修改某语言配置时不影响其他语言
- 可以轻松启用或禁用某种语言的支持
- 不同的用户可以根据需要自定义语言列表

## 动态加载

通过Neovim的`filetype`事件，当打开特定语言的文件时，对应的语言配置会自动加载和应用，用户无需手动激活。
