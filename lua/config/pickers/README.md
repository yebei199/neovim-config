# pickers - 选择器配置

本目录包含Neovim选择器(picker)的配置定义。选择器是一种交互式的列表和搜索界面，提高了文件、符号、缓冲区等查找操作的效率。

## 什么是选择器

选择器是一种用户交互界面，特点包括：
- **快速搜索** - 实时过滤和搜索列表内容
- **预览功能** - 选中项的实时预览（如文件内容、代码片段等）
- **键盘驱动** - 完全键盘操作，无需鼠标
- **可自定义** - 支持自定义外观、快捷键和行为

## 当前配置

### filetypes.lua
实现了文件类型选择器(FileType Picker)，该选择器：
- 列出所有已配置的编程语言
- 允许用户快速搜索和选择某种语言
- 显示该语言的配置状态
- 支持快速导航到语言配置文件

这个选择器对于需要频繁切换或查看不同语言配置的用户特别有用。

## 选择器库

项目通常使用Telescope作为选择器引擎，它提供了：
- **内置选择器** - 文件搜索、缓冲区列表、最近文件等
- **扩展接口** - 自定义选择器的开发接口
- **集成能力** - 与LSP、Git等工具的集成

## 使用场景

选择器在以下场景中提高工作效率：
- **快速打开文件** - 使用模糊搜索快速找到项目中的任何文件
- **代码导航** - 查找函数、类、符号等
- **历史导航** - 访问最近文件或编辑位置
- **内容搜索** - 在项目范围内搜索文本内容
- **快速操作** - 快速访问命令、快捷键帮助等

## 扩展指南

添加自定义选择器时的步骤：
1. 在本文件中创建选择器定义
2. 定义数据来源（文件系统、LSP、自定义函数等）
3. 配置搜索和过滤逻辑
4. 定义选中项的动作（打开文件、执行命令等）
5. 绑定快捷键以便快速访问

### 最佳实践
- 选择器应该响应快速，避免阻塞主线程
- 提供有意义的项目描述和预览
- 支持正则表达式或模糊搜索以提高查找效率
- 考虑为频繁使用的选择器提供快捷键
