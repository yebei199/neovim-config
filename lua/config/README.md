# config — 核心配置与初始化

Neovim 编辑器的初始化层，负责将选项、快捷键、事件处理、语言支持等各模块统一编排加载。此层直接操作 Neovim 全局状态，是插件加载前的基础准备阶段。

## 顶层文件

- `options.lua` — 全局编辑器选项，包括行号、缩进、搜索、折叠、滚动等基础设置
- `lazy.lua` — lazy.nvim 插件管理器的初始化，指定插件目录和加载策略
- `language.lua` — 语言配置系统的核心调度逻辑，按 filetype 事件加载对应语言模块
- `highlight.lua` — 在 colorscheme 加载后应用的自定义高亮覆盖
- `icons.lua` — 统一的图标常量定义，供状态栏、诊断、文件树等组件引用
- `neovide.lua` — 仅在 Neovide GUI 环境下生效的专属配置（字体、动画等）

## 子目录

**autocmds/** 集中管理所有自动命令，避免事件处理代码散落在各处。

**keymaps/** 按功能模块拆分快捷键，每个文件对应一个功能域（文件、Git、LSP、搜索、调试、开关、窗口、缩写），便于查找和修改，降低冲突风险。

**langs/** 存放在配置加载阶段（非 filetype 触发阶段）需要应用的语言相关设置，与根目录 `langs/` 配合构成完整的语言支持体系。

**pickers/** 定义超出 Telescope 内置能力的自定义选择器，例如按语言配置状态过滤并可跳转的 FileType Picker。
