# autocmds - 自动命令定义

本目录包含Neovim的自动命令(autocommand)定义。自动命令是一种事件驱动机制，在特定事件（如文件保存、缓冲区打开等）发生时自动执行相关操作，无需手动干预。

## 什么是自动命令

自动命令允许你在Neovim中建立"如果...则..."的规则。例如：
- 当用户打开特定文件类型时，自动加载相关工具和设置
- 当文件被修改时，自动执行格式化或校验
- 当编辑器退出时，自动清理临时文件或保存状态

这种事件驱动的方式使配置更加动态灵活，能够根据上下文自适应调整。

## 文件说明

### init.lua
主入口文件，组织和初始化所有自动命令定义。该文件：
- 导入其他自动命令模块
- 定义全局的自动命令组（用于后续管理）
- 确保自动命令的正确加载顺序

### quit_filetypes.lua
定义了某些特定文件类型在退出时的处理逻辑。例如：
- 关闭帮助文件窗口时的自动清理
- 临时缓冲区的自动删除
- 特殊文件类型的退出确认或状态保存

这个模块确保用户在处理特定文件类型时的体验一致性。

## 常见自动命令类型

项目中通常会使用以下自动命令：
- **FileType事件** - 根据文件类型自动配置选项和快捷键
- **BufRead/BufEnter事件** - 打开或进入缓冲区时的自动操作
- **BufWrite/BufWritePre事件** - 保存文件前的自动处理（如格式化）
- **User事件** - 自定义事件，由插件或配置主动触发
- **CmdlineEnter/CmdlineLeave事件** - 命令模式下的自动操作

## 自动命令的优势

相比在其他地方散落地编写初始化代码，使用自动命令提供了：
- **解耦性** - 事件处理与主配置逻辑分离
- **可维护性** - 集中管理所有事件处理规则
- **灵活性** - 支持复杂的条件判断和多步骤操作
- **性能** - 自动命令只在需要时执行，不浪费资源
