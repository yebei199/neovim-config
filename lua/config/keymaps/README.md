# keymaps - 快捷键映射定义

本目录集中管理Neovim的所有快捷键映射。通过模块化组织，使快捷键配置清晰易管理，也便于查找和修改特定功能的快捷键。

## 设计理念

快捷键按照功能分类存放，而非按照键盘位置或映射类型。这种方式的优势：
- **功能导向** - 查找某个功能的快捷键时直观快速
- **易于维护** - 修改某个功能的快捷键时只需改一个地方
- **易于扩展** - 添加新功能时清楚该在哪个文件添加
- **查重便利** - 容易发现和解决快捷键冲突

## 文件结构

### init.lua
主入口文件，负责：
- 导入各个分类的快捷键模块
- 定义全局快捷键（Leader等）
- 确保快捷键的正确加载顺序

### 功能分类文件

#### file.lua - 文件操作
- 打开、关闭、保存文件
- 文件浏览和导航
- 文件搜索和快速打开
- 最近文件列表

#### git.lua - Git集成
- 代码提交、推送、拉取
- 代码审查和差异查看
- 分支管理和切换
- 冲突解决辅助

#### lsp.lua - 语言服务器
- 转到定义、引用和实现
- 代码补全和代码片段
- 重命名和代码重构
- 诊断信息查看

#### search.lua - 搜索与替换
- 全局文本搜索
- 项目范围内搜索
- 搜索结果导航
- 高级搜索和替换

#### debug.lua - 调试功能
- 断点设置和管理
- 调试会话控制
- 表达式求值
- 堆栈和变量查看

#### toggle.lua - 功能开关
- 显示/隐藏元素（行号、相对行号等）
- 功能启用/禁用（拼写检查、格式化等）
- 编辑模式切换
- 视图配置快速切换

#### winbuf.lua - 窗口与缓冲区
- 窗口分割和关闭
- 窗口大小调整和移动
- 缓冲区导航和切换
- 窗口布局管理

#### abbr.lua - 文本缩写
- 常见单词的自动补全
- 代码片段的快速输入
- 符号和特殊字符的输入
- 自定义文本替换规则

## 快捷键约定

项目通常遵循的约定：
- **Leader键** - 通常设为空格或逗号，作为自定义快捷键的前缀
- **模式限定** - 不同模式（Normal、Insert、Visual等）的快捷键分别定义
- **符号含义** - 使用一致的快捷键符号（如`<leader>f`表示file相关操作）
- **冲突避免** - 避免与Neovim内置命令冲突，避免与插件快捷键冲突
