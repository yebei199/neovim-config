# keymaps — 快捷键映射

所有自定义快捷键的集中管理目录，按功能域拆分为独立文件，避免单文件过长、快捷键冲突难以排查的问题。

## 文件划分

- `init.lua` — 入口，设置 Leader 键并加载各子模块
- `file.lua` — 文件打开、保存、关闭及文件浏览相关操作
- `git.lua` — Git 提交、差异查看、blame、分支切换等 Git 集成操作
- `lsp.lua` — 跳转定义/引用、重命名、代码操作、悬停文档等 LSP 操作
- `search.lua` — 全局文本搜索、符号搜索、缓冲区内搜索替换
- `debug.lua` — 断点、调试会话控制、变量查看等 DAP 调试操作
- `toggle.lua` — 行号、相对行号、拼写检查、自动格式化等功能的即时开关
- `winbuf.lua` — 窗口分割、调整大小、缓冲区切换与关闭
- `abbr.lua` — Insert 模式文本缩写，减少常用词输入

## 设计意图

查找某个快捷键时可按功能域直接定位文件，而非在单一大文件中搜索。每个文件职责单一，添加新快捷键时明确该放入哪个文件。与插件配置中通过 `keys` 字段声明的懒加载快捷键形成互补——插件相关的核心激活键在插件 spec 中声明，通用操作键在此统一管理。
