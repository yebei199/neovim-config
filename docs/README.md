# docs — 项目文档与开发规划

本目录是项目知识与决策的集中管理处，存放两类文档：一是稳定的架构说明，二是过程性的功能开发规划。二者分工明确，前者描述"是什么"，后者记录"做了什么、为什么这样做"。

## 文件职责

**`architecture.md`** 描述整体代码架构，说明三层模块结构（config 初始化层、plugins 插件声明层、utils 工具库层）的职责划分与依赖方向，以及语言配置集中管理、Nix 版本锁定等核心设计决策。阅读此文件可以在不浏览具体代码的情况下理解整个配置系统的运行原理和设计意图。

**`keymap.md`** 汇总项目中所有自定义快捷键的映射关系与功能说明，按功能域分组（文件操作、Git、LSP、搜索、调试、窗口管理等），是快捷键查阅的参考文档，与 `lua/config/keymaps/` 中的具体实现保持同步。

**`plans/`** 子目录存放各阶段功能开发的设计文档与执行计划。每个计划文件以 `YYYY-MM-DD-<主题>` 命名，包含问题背景、方案对比、设计决策、实施步骤。该目录记录了开发过程中重要的架构演变，例如语言工具链重构、Rust 配置分离等重大变更的完整决策链，可作为后续维护时理解历史改动动机的参考。

## 与代码的关系

文档目录不包含可执行代码，其内容应与 `lua/`、`langs/`、`nix/` 等目录的实际实现保持同步。修改代码时若涉及架构层面的变动，需同步更新 `architecture.md`；新增或变更快捷键时需同步更新 `keymap.md`。
