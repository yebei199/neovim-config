# docs - 文档和项目规划

本目录存放Neovim配置项目的文档和实现规划，是项目知识和决策的集中管理地。

## 目录结构

- **architecture.md** - 项目整体架构设计文档，说明了项目的模块化结构、设计理念和核心组件的职责
- **plans/** - 实现计划和开发规划子目录，存放各个功能开发的设计方案、实现步骤和进度跟踪

## 主要内容

### 架构设计

architecture.md详细说明了项目的核心设计思想：
- 三层模块化结构：配置层（config）、插件层（plugins）、工具层（utils）
- 语言配置的统一管理方式，避免在多个文件中重复配置
- 插件的分类组织，按功能将插件分为组件、编辑器增强、LSP、UI和工具五大类
- Nix和lazy.nvim的集成方式，实现可复现的环境和声明式插件管理

### 开发规划

plans/目录包含各个阶段的实现计划，记录了新功能开发、系统重构、集成优化等工作的详细设计方案和执行步骤。这些文档帮助保证项目的方向一致性和质量标准。

## 使用指南

在开始重要的功能开发或架构调整前，应该先参考本目录的相关文档，了解项目的整体设计。实现完成后，如有新的设计决策或架构改进，应该更新文档以保持项目知识的完整性和准确性。
